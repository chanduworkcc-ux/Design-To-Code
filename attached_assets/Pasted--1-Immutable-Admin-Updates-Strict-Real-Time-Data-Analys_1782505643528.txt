## 2. Advanced Security, Fraud Mitigation, & Device Locking

* One Device, One Account Policy: Programmatically capture a unique device hardware fingerprint (using Canvas fingerprinting or a robust device info library) along with the user's current IP address upon registration and login.
* Registration/Login Blocking: Store these unique hardware signatures in a centralized device_registry collection. If an active device signature attempts to register a new account, or log into an alternate profile, block execution completely, reject the session, and display a "Device Blocked: Multi-Account Violation" warning.
* Strict Identity Check: Only one mobile number, one email account, one device footprint, and one account per individual user are permitted.
* Admin Control Bans: Provide the Admin with two distinct permanent termination vectors from the Admin panel:
* Account Ban: Revokes Firebase Auth access tokens immediately and marks the user's database status as banned.
   * Device Footprint Ban: Blacklists the target device fingerprint and IP address globally inside Firestore rules, permanently locking that hardware out of the registration or login screens.
* Temporary Suspension Engine: Admin can suspend any user for a specific duration (e.g., a custom number of days set by a text/number field). When a suspended user authenticates, show a locked, full-screen countdown screen displaying the remaining days, hours, and minutes until access is restored.

## 3. Real-Time Heartbeats, Audit Logs, & Analytics

* Online/Offline Pulse Tracking: Integrate the Firebase Realtime Database .info/connected status network.
* When a user connects, set /status/{uid} = { state: 'online', last_changed: serverTimestamp() }.
   * On disconnect (onDisconnect()), flip the value to 'offline'.
   * The Admin dashboard user management view must listen to this endpoint in real time and render a dynamic green badge for Online users and a grey badge for Offline users.
* Live User Analytics & System Logs: Every interactive action performed by a user (e.g., placing an order, viewing a page, updating profile options, generating a referral link, triggering a payout request) must write a structured event object into a global customer_logs collection.
* Log Parameters: Each log must capture timestamp, user_id, action_type, ip_address, device_fingerprint, and payload_details. The Admin panel must feature a live scrolling monitor of these analytical logs.
